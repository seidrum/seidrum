# Agent Skills

Behavioral RAG for agents — skills are reusable behavioral recipes that agents discover by semantic similarity and load on demand.

## How It Works

Skills are short instruction snippets stored with vector embeddings. When an agent processes an event, the system embeds the current context and retrieves the most relevant skills. The agent also has a built-in capability to explicitly search and load skills.

### Two-Layer Retrieval

**Layer 1: Vector embeddings** — each skill's description is embedded and indexed. When an event arrives, the system computes semantic similarity between the event context and all skill embeddings, returning the top-K most relevant.

**Layer 2: Short instruction snippets** — the actual behavioral guidance. Not full documentation — just a concise instruction that gets injected into the LLM prompt.

```
Event arrives → enhance query with conversation context
  → embed enhanced query
  → vector search against skills collection
  → retrieve top-K matching snippets
  → inject into prompt between system prompt and identity prompt
```

### Prompt Assembly with Skills

```
[System prompt]        ← platform awareness (immutable)
[Active skills]        ← dynamically retrieved + explicitly loaded snippets
[Identity prompt]      ← agent personality (from YAML)
[Context]              ← facts, tasks, conversation history
[User message]
```

## Skill Document

```json
{
  "_key": "skill_code_review",
  "id": "code-review",
  "description": "Reviewing code changes for bugs, security issues, and style problems",
  "snippet": "When reviewing code: check for OWASP top 10 vulnerabilities, missing error handling, test coverage gaps, and performance implications. Report each issue with file, line, severity, and recommended fix.",
  "embedding": [0.12, -0.34, ...],
  "source": "system",
  "scope": null,
  "tags": ["code", "review", "security"],
  "created_at": "2026-03-24T10:00:00Z",
  "learned_from": null
}
```

| Field | Purpose |
|-------|---------|
| `description` | What triggers retrieval — this text gets embedded for similarity search |
| `snippet` | What gets injected into the prompt — the actual behavioral instruction |
| `embedding` | Vector from the description (generated by the embedding service) |
| `source` | How this skill was created: `system`, `user`, `learned`, `agent` |
| `scope` | Optional scope filter — only retrieve for agents in matching scopes |
| `tags` | Optional tags for explicit search |
| `learned_from` | Conversation ID where this skill was learned (for provenance) |

## Sources of Skills

### System Skills (deploy-time)

YAML files in `skills/` directory, embedded and indexed on kernel startup:

```yaml
# skills/code-review.yaml
skill:
  id: code-review
  description: "Reviewing code changes for bugs, security issues, and style problems"
  snippet: |
    When reviewing code: check for OWASP top 10 vulnerabilities, missing error
    handling, test coverage gaps, and performance implications. Report each issue
    with file, line, severity, and recommended fix.
  tags: [code, review, security]
```

### User Instructions (conversational)

The user tells the agent a behavioral preference:

```
User: "From now on, always format financial numbers with two decimal places and currency symbol"

Agent recognizes this as behavioral instruction → stores as skill:
  description: "formatting financial numbers and currency values"
  snippet: "Always format financial numbers with 2 decimal places and currency symbol (e.g., $1,234.56)"
  source: "user"
```

Next time the agent encounters financial data, the embedding search finds this skill automatically.

### Learned Behavior (agent self-teaches)

After completing a complex task successfully, the agent extracts reusable patterns:

```
Agent completed a database migration (12 tool calls, 45 minutes).

Inner monologue: "This was successful. The critical steps were: backup first,
verify schema compatibility, run in a transaction, verify data integrity after.
I should save this for next time."

Agent stores skill:
  description: "database migration procedure and best practices"
  snippet: "For database migrations: 1) backup first 2) verify schema compatibility
    3) run in a transaction 4) verify data integrity after 5) update application
    config 6) test critical queries"
  source: "learned"
  learned_from: "conv_01JXYZ..."
```

### Agent-to-Agent Knowledge

When an agent discovers something useful, it can share it as a skill:

```
Research Agent discovers that v2 API is 3x faster for bulk operations.

Stores skill:
  description: "API version selection for bulk vs single operations"
  snippet: "Use v2 API for bulk operations (3x faster than v1). v1 is fine for single-item requests."
  source: "agent:research-agent"
```

This skill is now available to all agents via embedding search.

## Packaged Skills

Skills can be distributed as installable packages:

```bash
seidrum skill install code-review
seidrum skill install seidrum-community/deploy-monitor
seidrum skill list
seidrum skill remove deploy-monitor
```

A package is a directory or archive containing one or more skill YAML files. On install:
1. Copy YAML files to `skills/` directory
2. Embed each skill's description
3. Store in the `skills` ArangoDB collection
4. Available immediately for retrieval

## Query Enhancement

The embedding search query is not just the raw user message — it's enhanced with conversation context for better retrieval:

```
Raw message: "Can you fix the test?"

Enhanced query (using recent conversation + active tasks):
"Fix a failing Python unit test in the auth module.
Context: JWT migration, pytest, token expiration edge cases."
```

Enhancement sources:
- Current message text
- Last 3-5 conversation messages (topic context)
- Active tasks (what the agent is working on)
- Current scope name

This can be done by concatenating recent messages before embedding, or by a lightweight LLM call to summarize the query context.

## Conversation Continuity

When a skill is retrieved for a conversation turn, the conversation message is tagged with the active skills:

```json
{
  "role": "assistant",
  "content": "I'll review this code...",
  "active_skills": ["code-review", "security-checklist"]
}
```

On the next turn, the system sees "this conversation has been using code-review" and includes it in the context, even if the new message alone wouldn't retrieve it. The agent is told:

```
Previously active skills in this conversation: code-review, security-checklist
```

This prevents mid-conversation skill drops that could lead to inconsistent behavior.

If the agent no longer needs a skill, it can explicitly unload it:

```
tool_call: manage-skills
arguments: { "action": "unload", "skill_id": "code-review" }
```

## Built-in Capabilities

| Capability | Description |
|-----------|-------------|
| `search-skills` | Search skills by semantic query. Returns matching skill IDs, descriptions, and snippets. |
| `load-skill` | Explicitly load a skill by ID into the current conversation context. |
| `manage-skills` | Unload a skill, or list currently active skills. |
| `save-skill` | Create a new skill from the current conversation (for learned behavior). |

### `search-skills`

```
tool_call: search-skills
arguments: { "query": "how to handle database migrations safely" }

Result: [
  { "id": "database-migration", "description": "...", "snippet": "...", "score": 0.91 },
  { "id": "rollback-procedures", "description": "...", "snippet": "...", "score": 0.78 }
]
```

### `load-skill`

```
tool_call: load-skill
arguments: { "skill_id": "database-migration" }

Result: { "loaded": true, "snippet": "For database migrations: 1) backup first..." }
```

The skill's snippet is injected into the prompt for the remainder of the conversation.

### `save-skill`

```
tool_call: save-skill
arguments: {
  "description": "database migration procedure and best practices",
  "snippet": "For database migrations: 1) backup first 2) verify schema...",
  "tags": ["database", "migration", "ops"]
}

Result: { "saved": true, "skill_id": "skill_01JXYZ" }
```

## Scaling to Thousands of Skills

The two-layer system handles scale naturally:

- **Embedding index** — ArangoDB ANN (approximate nearest neighbor) index on the `embedding` field. Vector similarity search is O(log n), so 1,000 or 10,000 skills is fine.
- **Top-K retrieval** — only the top 3-5 most relevant skill snippets are injected. The LLM never sees all skills.
- **Scope filtering** — skills can be scoped. An agent in `scope_projects` only searches project-scoped skills + global skills. This reduces the search space.
- **Snippet budget** — skill snippets compete for prompt token budget alongside facts, tasks, and history. Short, focused snippets win.

## Infrastructure Dependencies

The skills system requires:

1. **Embedding service** — generates vectors from text (OpenAI, Gemini, or local model)
2. **Vector index** — ArangoDB ANN index on the `skills` collection
3. **Skills collection** — ArangoDB vertex collection storing skill documents
4. **Retrieval step in consciousness loop** — between event receipt and LLM call

The embedding service is foundational — it also powers RAG for conversations, entity search, and content retrieval.

## New ArangoDB Collection

```
skills (vertex collection)
├── _key: ULID
├── id: human-readable identifier
├── description: text (embedded for similarity search)
├── snippet: text (injected into prompt)
├── embedding: array of floats
├── source: "system" | "user" | "learned" | "agent:{id}"
├── scope: optional scope filter
├── tags: array of strings
├── created_at: datetime
├── learned_from: optional conversation_id
└── indexes:
    ├── persistent: [id] (unique)
    ├── persistent: [source]
    └── vector: [embedding] (ANN index)
```
